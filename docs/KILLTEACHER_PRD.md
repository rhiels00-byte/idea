# Killteacher PRD

## 1. 제품 개요

### 제품명
Killteacher

### 한 줄 설명
선생님이 과목과 주제를 입력하면 AI가 실제 수업용 문항을 만들고, 학생들이 레이드 게임처럼 문제를 풀며 선생님을 쓰러뜨리는 실시간 수업 참여 서비스.

### 제품 목적
- 수업 도입과 정리 시간을 더 몰입감 있게 만든다.
- 문제 풀이를 게임화해 학생 참여도를 높인다.
- 선생님이 별도 제작 도구 없이 바로 문항을 생성하고 진행할 수 있게 한다.

### 핵심 가치
- AI 기반 빠른 문항 생성
- 선생님용/학생용 화면 분리
- 선생님 HP, 타격 로그, 랭킹 중심의 게임형 수업 경험
- 수업 종료 후 기여도 기반 칭호와 뱃지 제공

## 2. 타깃 사용자

### 주요 사용자
- 초등, 중등, 고등 선생님
- 수업에 참여하는 학생

### 사용 상황
- 단원 도입 퀴즈
- 형성평가
- 복습 수업
- 팀 기반 참여형 활동

## 3. 사용자 문제

### 선생님 관점
- 수업 전에 퀴즈를 만드는 데 시간이 오래 걸린다.
- 학생 참여를 끌어내기 어렵다.
- 문제 풀이 결과를 한눈에 보여주기 어렵다.

### 학생 관점
- 일반 퀴즈는 지루하게 느껴질 수 있다.
- 내 기여도가 드러나지 않는다.
- 수업 참여의 재미 요소가 부족하다.

## 4. 제품 목표

### 목표
- 선생님이 1분 이내에 레이드 퀴즈 세션을 만들 수 있게 한다.
- 학생이 입장 후 바로 게임 맥락을 이해할 수 있게 한다.
- 문제 풀이 결과가 즉시 HP, 공격 로그, 랭킹에 반영되게 한다.

### 성공 기준
- 선생님이 과목과 주제 입력 후 문항 생성 가능
- 학생 최대 5명 입장 가능
- 선생님 시작 시 학생 화면도 자동 전환
- 정답 제출 시 HP 감소, 로그, 랭킹 반영
- 종료 후 칭호와 뱃지 생성

## 5. 핵심 사용자 흐름

### 선생님 흐름
1. 선생님이 학교급, 학년, 학급명, 과목, 주제/키워드, 문항 수를 입력한다.
2. AI 문항 생성 버튼을 눌러 선생님 캐릭터와 문제를 만든다.
3. QR 코드 또는 학생 링크를 통해 학생을 입장시킨다.
4. 학생 수를 확인하고 게임을 시작한다.
5. 문제 진행 중 현재 문항, HP, 타격 로그, 실시간 랭킹을 확인한다.
6. 게임 종료 후 최종 랭킹과 칭호/뱃지를 확인한다.

### 학생 흐름
1. 학생이 링크 또는 QR로 접속한다.
2. 닉네임과 이모지 캐릭터를 선택한다.
3. 대기실에 입장한다.
4. 선생님이 시작하면 게임 화면으로 전환된다.
5. 문항을 풀고 정답 제출 시 공격 결과를 본다.
6. 종료 후 개인 기여도, 칭호, 뱃지를 확인한다.

## 6. 주요 기능 요구사항

### 6.1 AI 문항 생성
- 입력값: 학교급, 학년, 학급명, 과목, 주제/키워드, 문항 수
- 입력 순서: 학교급 → 학년 → 학급명 → 과목 → 주제 → 문항 수
- Gemini 2.5 Flash API를 사용해 문항과 선생님 캐릭터 정보를 생성한다.
- 어떤 주제/키워드든 AI가 해당 주제에 맞는 4지선다형 문항을 만든다.
- 난이도는 학교급에 따라 자동 조절한다 (초등: 쉽게, 중등: 보통, 고등: 어렵게).
- Gemini 호출 실패 시 fallback 문항을 생성한다.
- fallback은 산술형 키워드(덧셈, 뺄셈, 곱셈, 나눗셈)를 인식하여 실제 계산 문제를 만든다.

### 6.2 게임 세션 생성
- 세션 ID를 URL query parameter로 관리한다.
- 선생님 화면과 학생 화면이 같은 세션을 공유한다.
- `localStorage`와 `BroadcastChannel`로 상태를 동기화한다.

### 6.3 선생님용 화면
- 입력 폼 제공 (학교급 → 학년 → 학급명 순서)
- QR 코드 제공
- 학생 참여 현황 표시
- 현재 문제, 선생님 연출, HP 바, 타격 로그, 실시간 랭킹 표시
- 데모 학생 5명 자동 채우기 기능 제공
- 다음 문항으로 이동 버튼 (수동 스킵)

### 6.4 학생용 화면
- 닉네임 입력
- 이모지 캐릭터 선택
- 대기실 화면
- 문제 풀이 화면
- 실시간 랭킹 확인
- 종료 후 개인 칭호 및 뱃지 확인
- 관전 모드: 직접 입장하지 않은 학생도 PLAY/RESULT 화면 열람 가능

### 6.5 선생님 캐릭터 연출
- 학교급에 따라 서로 다른 영상 사용
  - `public/videos/elementary.mp4` (초등)
  - `public/videos/middle.mp4` (중등)
  - `public/videos/high.mp4` (고등)
- 비디오 세그먼트 제어 방식으로 애니메이션 상태를 관리한다.
  - idle: 대기 루프 (캐릭터가 가만히 움직이는 모습)
  - hitLight: 가벼운 피격 (데미지 < 15)
  - hitBig: 강한 피격 (데미지 >= 15) → recover → idle
  - recover: 회복 후 idle 복귀
- 피격 시 데미지 숫자가 부유하며 사라지는 애니메이션 제공
- HP 바는 영상 밖 상단에 별도 배치

### 6.6 랭킹 및 보상
- 누적 데미지, 정답 수, 정확도를 기준으로 랭킹 계산
- 게임 종료 후 Gemini로 학생별 칭호와 뱃지 생성
- 실패 시 fallback 칭호/뱃지 사용

## 7. 게임 규칙

### 7.1 문항 진행
- 문항이 N개라면, N개가 모두 소진되면 게임이 종료된다.
- 선생님이 "다음 문항으로" 버튼을 누르는 것도 문항 소진에 포함된다.

### 7.2 1개 문항을 풀 때
- 학생 1명당 응시 기회는 **1번**이다.
- 정답이 나올 때까지 선생님 캐릭터는 idle 상태를 유지한다.
- **1명이라도 먼저 정답을 맞히면**:
  - hit 애니메이션 효과가 발생한다.
  - 선생님 HP가 감소한다.
  - 자동으로 다음 문항으로 넘어간다.
- **모든 학생이 오답이라면**:
  - "공격 실패!" 애니메이션 효과가 나온다 (빨간 텍스트 + 흔들림).
  - 선생님 HP는 깎이지 않는다.
  - 자동으로 다음 문항으로 넘어간다.

### 7.3 다음 문항으로 넘어갈 때
- 선생님 캐릭터는 다시 idle 모습으로 돌아간다.

### 7.4 선생님이 "다음 문항으로" 수동 스킵할 때
- 선생님 HP는 깎이지 않는다.
- 문항 소진에 포함된다.

### 7.5 게임 종료 조건
- 모든 문항이 소진되었을 때
- 선생님 HP가 0 이하가 되었을 때

### 7.6 게임 결과
- **HP <= 0**: "선생님 처치 성공!" (Raid Clear)
- **HP > 0**: "선생님이 살아남았다!" (Raid Failed)
- 처치 실패여도 학생별 기여도, 칭호, 뱃지는 그대로 노출한다.

### 7.7 데미지 계산
- 정답 시 기본 데미지: 12
- 연속 정답 보너스: streak × 3 (최대 4연속, +12)
- 최대 단일 데미지: 12 + 12 = 24

## 11. 파일 구조와 역할

### 진입 파일
- `src/main.jsx` — 앱 entry point. `Killteacher`만 렌더링.
- `src/Killteacher.jsx` — 선생님/학생 화면 분기, 세션 ID 생성 및 URL 동기화, 공용 상태 훅 연결.

### 선생님/학생 화면
- `src/killteacher/TeacherRaidScreen.jsx` — 선생님용 메인 UI (문항 생성 폼, QR, 학생 목록, 아레나, 타격 로그, 랭킹, 결과).
- `src/killteacher/StudentRaidScreen.jsx` — 학생용 메인 UI (닉네임/이모지 선택, 대기실, 문제 풀이, 랭킹, 관전 모드, 결과).

### 게임 로직
- `src/killteacher/useRaidSession.js` — 세션 전체 상태 정의, 학생 입장, 데미지 계산, 문제 진행, HP, 로그/랭킹, 데모 학생 채우기, 종료 보상, 공격 실패 감지.

### AI 로직
- `src/killteacher/gemini.js` — Gemini API 호출, 문항 생성 프롬프트, 칭호/뱃지 생성, fallback 문항(산술형 포함).

### 선생님 캐릭터 연출
- `src/killteacher/BossCanvas.jsx` — 비디오 세그먼트 제어 컴포넌트. idle/hitLight/hitBig/recover 상태 전환, 데미지 숫자 부유, 공격 실패 텍스트 + 흔들림 효과, HP 바.

### 스타일/정적 파일
- `src/index.css` — 전역 스타일 진입점.
- `public/videos/elementary.mp4` — 초등 선생님 캐릭터 영상.
- `public/videos/middle.mp4` — 중등 선생님 캐릭터 영상.
- `public/videos/high.mp4` — 고등 선생님 캐릭터 영상.

### 참고/아카이브
- `docs/KILLTEACHER_PRD.md` — 이 문서.
- `archive/legacy-ui/` — 이전 UI 코드 아카이브.
